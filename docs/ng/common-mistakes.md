# よくある失敗事例と対策

実際の現場で起きた失敗事例から学びます。**同じミスを繰り返さないために**。

!!! danger "重要"
    これらの事例は実際に起きた失敗です。「自分は大丈夫」と思わず、対策を学びましょう。

## 🚨 重大NG事例（躯体工事）

### 事例1: スリーブと主筋の干渉 💰 損失: 100万円

<div class="ng-case">

**状況**:  
設備配管用のスリーブを梁に設置。コンクリート打設後、スリーブが梁の主筋と干渉していることが発覚。

**結果**:  
- コンクリートをコア抜きして配管を通す
- 構造補強が必要（鉄板接着）
- 工期2週間遅延
- コスト追加100万円

**原因**:  
- BIMチェックを実施しなかった（「時間がない」と省略）
- 躯体図と設備図を重ねずに目視確認だけ
- 「たぶん大丈夫」という楽観視

**対策**:  
✅ スリーブ設置前にBIMで干渉チェック必須  
✅ 躯体図と設備図を必ず重ね合わせる  
✅ 疑わしい箇所は必ず確認  

**講師コメント**:  
「時間がない」は最も危険な言葉です。BIMチェック30分で100万円の損失を防げました。

</div>

### 事例2: 梁せいの入力ミス 💰 損失: 200万円

<div class="ng-case">

**状況**:  
BIMモデルで梁のせい（高さ）を600mmと入力すべきところ、500mmと入力。干渉チェックで「問題なし」と報告。

**結果**:  
- 実際には天井内でダクトと梁が干渉
- ダクトルート全変更
- 天井高さが100mm低くなった
- 工期1ヶ月遅延、コスト200万円追加

**原因**:  
- 施工図とBIMモデルの寸法照合を省略
- 「図面通りに入力したはず」という思い込み
- ダブルチェック（第三者確認）がなかった

**対策**:  
✅ モデル作成後、施工図と寸法を必ず照合  
✅ 重要寸法（梁せい、柱断面など）はダブルチェック  
✅ 「入力したはず」ではなく「確認した」という文化  

**講師コメント**:  
入力ミス1つで200万円。BIMは「正しい情報を入れれば正しい結果が出る」ツールです。

</div>

---

### 事例3: 古い施工図でBIMモデル作成 💰 損失: 150万円

<div class="ng-case">

**状況**:  
施工図が改訂されたが、古い版（Rev.1）でBIMモデルを作成。最新版（Rev.3）との差異に気づかず施工開始。

**結果**:  
- 柱位置が200mmずれていた
- 配筋完了後に発覚
- 鉄筋を全てやり直し
- 工期2週間遅延、コスト150万円

**原因**:  
- 図面の版管理がされていなかった
- BIMモデルに「どの版の図面を使用したか」の記録がない
- 施工図とBIMモデルの版番号照合をしなかった

**対策**:  
✅ 図面の版番号・日付を常に確認  
✅ BIMモデルに「使用図面の版番号」を記録  
✅ 施工前に最新図面との照合を必須化  

**講師コメント**:  
図面管理は地味ですが超重要。「最新版を使う」は鉄則です。

</div>

## 🔧 設備工事のNG事例

### 事例4: 天井高さ不足 💰 損失: 300万円

<div class="ng-case">

**状況**:  
BIMでダクト・配管を配置したが、天井高さの確認を忘れた。仕上げ工事で天井が設計より200mm低いことが判明。

**結果**:  
- 居室が圧迫感のある空間に
- 発注者から「設計通りの天井高さにして」と指摘
- ダクト・配管を全て移動
- 工期1ヶ月遅延、コスト300万円

**原因**:  
- 「天井高さ」の確認項目がなかった
- ダクト・配管の高さだけ確認し、天井との関係を見なかった
- 設計図の天井高さを確認しなかった

**対策**:  
✅ BIMモデルに「設計天井高さ」を明示  
✅ ダクト・配管配置後、必ず天井高さを確認  
✅ 最低天井高さを守ることをチェックリスト化  

**講師コメント**:  
天井高さは居住性に直結。「ダクトが入った」だけでは不十分です。

</div>

---

### 事例5: 排水勾配の確保ミス 💰 損失: 80万円

<div class="ng-case">

**状況**:  
BIMで排水配管をモデル化したが、勾配（1/100）を確認しなかった。施工後、水が流れないことが判明。

**結果**:  
- 配管をすべてやり直し
- 天井を一部解体
- コスト80万円、工期1週間遅延

**原因**:  
- 排水配管に勾配が必要という知識不足
- BIMモデルで勾配を設定しなかった
- 「水平配管でも大丈夫だろう」という誤解

**対策**:  
✅ 排水配管は必ず勾配（1/100以上）を設定  
✅ BIMモデルで配管の高さを始点・終点で確認  
✅ 施工図の勾配記号を見逃さない  

**講師コメント**:  
排水は重力で流します。勾配は絶対条件です。

</div>

## 🏗️ 施工計画のNG事例

### 事例6: クレーン旋回範囲が敷地外 💰 損失: 500万円

<div class="ng-case">

**状況**:  
タワークレーンを配置したが、BIMで旋回範囲を確認しなかった。試運転で隣接建物に旋回範囲が出ることが判明。

**結果**:  
- クレーンを移設（基礎からやり直し）
- 工程が1ヶ月遅延
- クレーン会社から違約金請求
- コスト500万円

**原因**:  
- タワークレーンの旋回範囲を3Dで確認しなかった
- 隣接建物の高さを考慮しなかった
- 「敷地内だから大丈夫」という思い込み

**対策**:  
✅ タワークレーンの旋回範囲を3Dで必ず確認  
✅ 隣接建物・道路・架空線との関係を確認  
✅ 仮設計画はBIMで事前検証必須  

**講師コメント**:  
クレーン事故は死亡事故にもつながります。必ず3Dで確認を。

</div>

### 事例7: 足場が不足 💰 損失: 120万円

<div class="ng-case">

**状況**:  
仮設計画で足場範囲をBIMで確認しなかった。外壁工事で足場が足りず、作業ができないことが判明。

**結果**:  
- 足場を追加設置
- 外壁工事が2週間遅延
- コスト120万円

**原因**:  
- 足場範囲を2D図面だけで確認
- 複雑な形状の建物で見落とし
- 「だいたいで大丈夫」という甘い判断

**対策**:  
✅ 足場範囲を3Dで可視化  
✅ 作業範囲と足場範囲を重ね合わせて確認  
✅ 複雑な形状は特に注意  

</div>

## 📊 BIM運用のNG事例

### 事例8: モデルが重すぎて開かない 💰 損失: 時間

<div class="ng-case">

**状況**:  
すべての部材を詳細にモデル化（ボルト1本まで）。ファイルサイズが5GBを超え、パソコンで開けなくなった。

**結果**:  
- 干渉チェックに10時間かかる
- モデルを簡略化し直す作業に1週間
- 作業効率が大幅低下

**原因**:  
- 「詳細にすれば良い」という誤解
- LOD（詳細度）の概念を理解していない
- 目的に応じたモデリングをしなかった

**対策**:  
✅ LOD（詳細度）を理解し、目的に応じたモデリング  
✅ 施工段階はLOD300-400が適切  
✅ ボルト1本までモデル化する必要はない  

**講師コメント**:  
「詳細=良い」ではありません。目的に合った精度が重要です。

</div>

---

### 事例9: データ共有ルールなし 💰 損失: 時間

<div class="ng-case">

**状況**:  
元請・設備会社・鉄骨会社がそれぞれ別のBIMソフトでモデル作成。データが統合できず、干渉チェックができない。

**結果**:  
- データ変換に毎回2日かかる
- 変換ミスで干渉が見逃される
- 結局、手作業で確認

**原因**:  
- BIMソフトの統一ルールがなかった
- データ形式（IFC、DWGなど）を決めていなかった
- 「後で何とかなる」という甘い見通し

**対策**:  
✅ 事前にBIMソフト・データ形式を統一  
✅ IFC形式での受け渡しルールを決定  
✅ BIM実行計画書（BEP）を作成  

</div>

---

### 事例10: BIMモデルと現場が違う 💰 損失: 信頼

<div class="ng-case">

**状況**:  
施工中に変更が発生したが、BIMモデルを更新しなかった。最終的にBIMモデルと実際の建物が全く違う状態に。

**結果**:  
- 竣工図作成に膨大な時間
- BIMモデルが信用されなくなった
- 「BIMは使えない」という評価

**原因**:  
- 変更があってもBIMを更新しない文化
- 「図面だけ直せば良い」という考え
- BIMモデルの維持管理ルールがない

**対策**:  
✅ 変更があったら図面とBIMモデル両方を更新  
✅ As-Builtモデル（竣工モデル）の作成ルール化  
✅ BIM更新の責任者を明確化  

**講師コメント**:  
BIMモデルが現場と違うなら、BIMを使う意味がありません。

</div>

## 📋 NG事例から学ぶチェックリスト

### 施工前の必須確認

- [ ] 使用する図面の版番号・日付を確認
- [ ] BIMモデルの寸法を施工図と照合
- [ ] 重要寸法（梁せい、柱断面など）をダブルチェック
- [ ] 躯体と設備の干渉チェック
- [ ] 天井高さの確保を確認
- [ ] 排水配管の勾配を確認
- [ ] タワークレーンの旋回範囲を確認
- [ ] 足場の作業範囲を確認

### BIM運用の必須ルール

- [ ] LOD（詳細度）を目的に応じて設定
- [ ] データ形式を統一（IFCなど）
- [ ] 変更時は図面とBIM両方を更新
- [ ] BIMモデルに使用図面の版番号を記録
- [ ] 定期的にモデルと現場を照合

## 💡 失敗しないための心構え

### 1. 「時間がない」は最も危険な言葉
時間がないからこそ、BIMで事前チェックが重要です。

### 2. 「たぶん大丈夫」は禁句
疑わしい箇所は必ず確認。後で直すのは10倍コストがかかります。

### 3. 施工図が先、BIMは後
BIMは施工図の品質以上にはなりません。

### 4. 記録を残し、次に活かす
失敗は恥ずかしいことではありません。記録して次に活かしましょう。

### 5. チーム全体で情報共有
一人が知っていても意味がありません。チーム全体で共有しましょう。
